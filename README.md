# Retail Store Bank Statement Analysis

## Project Overview
This project analyzes the bank statement of a retail store by categorizing transactions into different accounting heads. At the end of the analysis, a daily report is prepared showing the financial summary for decision-making purposes.

## Tools Used
- Excel (for data cleaning and categorization)
- Python (Pandas & Matplotlib for analysis and visualization)
- Power BI / Tableau (optional for dashboards)

## Data Cleaning & Preparation
- Imported bank statement in Excel format
- Standardized date and amount formats
- Categorized each transaction into accounting heads such as:
  - Sales
  - Expenses
  - Purchases
  - Bank charges
  - Other income
- Checked for missing or duplicate entries

## Analysis & Reporting
- Calculated total amounts for each accounting head per day
- Identified trends and anomalies in transactions
- Generated end-of-day summary report including:
  - Total sales
  - Total expenses
  - Net cash flow
  - Alerts for unusual transactions

## Key Insights (Example)
- Sales transactions account for 65% of total daily inflows
- Expenses are highest on weekends due to supplier payments
- Bank charges are consistent, averaging $25 per day
- Some transactions were uncategorized and need review

## Recommendations
- Automate categorization for recurring transactions to save time
- Monitor high-value transactions for potential errors
- Use dashboards to visualize trends over weeks/months
